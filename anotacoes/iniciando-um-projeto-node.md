# Iniciando um projeto Node.js

## O que é o Node.js

O JavaScript pode ser executado em dois tipos de ambientes:

- **Navegadores**: Chrome, Firefox (Mozilla), Edge, entre outros.
- **Runtimes (ambientes de execução)**: Node.js, Deno, Bun etc.

No backend, utilizamos principalmente o **Node.js**, pois ele é o runtime mais antigo, mais estável e, por isso, o mais adotado pelo mercado.

---

## O que é um ambiente de execução (runtime)?

Um ambiente de execução é o local onde o código JavaScript realmente roda.  
Ele já vem com várias funcionalidades prontas, que usamos desde o início dos estudos.

Um exemplo clássico é o `console.log()`.

A biblioteca `console` e o método `log` são tão comuns que já vêm implementados por padrão tanto nos navegadores quanto no Node.js.

Isso mostra que um runtime não é apenas o motor que executa o código, mas também um conjunto de **bibliotecas nativas** que facilitam o desenvolvimento.

---

## O que é uma biblioteca?

Uma biblioteca é um conjunto de códigos reutilizáveis criado para resolver um problema ou atender a uma necessidade específica.

Na programação, muitos problemas se repetem. Por exemplo:

- Exibir mensagens no console
- Manipular datas
- Trabalhar com arquivos
- Fazer requisições HTTP

Essas tarefas são tão comuns que não faz sentido escrever tudo do zero.  
Em vez disso, reutilizamos código pronto, evitando “reinventar a roda”.

---

## Por que utilizamos bibliotecas?

- Existem bibliotecas para praticamente qualquer necessidade
- Aumentam a produtividade e a eficiência
- São mantidas por empresas e pela comunidade
- Qualquer pessoa pode criar uma biblioteca e publicá-la para outras pessoas utilizarem

Em resumo, bibliotecas nos ajudam a escrever menos código, com mais qualidade e em menos tempo.
